# AUR packaging for himmelblau

This directory holds the **source-of-truth** PKGBUILD template and install
scriptlet that CI publishes to the AUR.

## Files

| File | Status | Purpose |
|------|--------|---------|
| `PKGBUILD.in` | committed | PKGBUILD template with `@PKGVER@` and `@SHA256@` sentinels |
| `himmelblau.install` | committed | pacman install scriptlet (post-install / upgrade / remove messages) |
| `.SRCINFO` | **never committed** | regenerated by `makepkg --printsrcinfo` at publish time |
| `PKGBUILD` | **never committed** | rendered from `PKGBUILD.in` at publish time |

## How a release is published

1. Upstream tags a release (e.g. `3.1.5`).
2. `.github/workflows/aur-publish.yml` renders `PKGBUILD` from `PKGBUILD.in`:
   ```bash
   sed -e "s|@PKGVER@|${version}|g" \
       -e "s|@SHA256@|${sha256}|g" \
       platform/arch/PKGBUILD.in > /tmp/aur-work/PKGBUILD
   ```
3. `makepkg --printsrcinfo > .SRCINFO` against the rendered PKGBUILD.
4. `namcap PKGBUILD` validates. (`namcap` does not lint `.SRCINFO` directly; lint the built `.pkg.tar.zst` if you want post-build validation.)
5. The deploy action commits + pushes to `ssh://aur@aur.archlinux.org/himmelblau.git`.

## Editing this template

- Hand-edit `PKGBUILD.in` directly. No code generator. No Python.
- Bump `pkgrel` if you make a packaging-only change against the same upstream
  version. The release-driven CI flow always resets `pkgrel=1` for new
  upstream versions.
- If you change `license`, `depends`, `provides`, etc., also update the
  rationale comment near the field. Future-you and reviewers will thank you.

## Why a template instead of a generator?

AUR convention favors checked-in PKGBUILDs, an audit on a previous Python
generator surfaced incorrect install lines that survived multiple review
passes, and the only field a generator would actually derive (`pkgver`) is
already derived from the GitHub release tag in CI.

## Local testing

Run the podman-based validation harness:

```bash
./platform/arch/test/lint.sh                  # lint, ~30s (latest local tag)
./platform/arch/test/lint.sh --version 3.1.5  # lint, explicit version
./platform/arch/test/lint.sh --build          # full makepkg build (20–40 min under qemu)
```

See `platform/arch/test/README.md` for the full usage, exit-code semantics,
and the Apple Silicon / rootless-podman caveats baked into the script.
